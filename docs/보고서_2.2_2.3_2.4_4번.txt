■ 기술 1 — SSE(Server-Sent Events) fetch 스트리밍 기반 실시간 이벤트 수신

  배경 및 선택 이유
  서버에서 발생하는 위험 이벤트(낙상, 장기 무활동 등)를 보호자 화면에
  즉시 전달하기 위해 실시간 푸시 기술이 필수적이었다. 일반적으로
  SSE(Server-Sent Events) 구현에 브라우저 내장 EventSource API를
  사용하지만, EventSource는 HTTP 헤더를 커스터마이징할 수 없어
  JWT 기반의 Authorization 헤더를 함께 전송하는 것이 불가능하다.
  이 제약을 극복하기 위해 fetch API의 ReadableStream을 직접 활용하는
  방식으로 저수준 SSE 클라이언트를 직접 구현하였다.

  구현 내용
  fetch로 /api/v1/alerts/stream에 GET 요청을 보내면서
  Authorization: Bearer <token> 헤더를 함께 전송한다.
  응답 body에서 getReader()로 ReadableStream을 취득하고,
  청크 단위로 UTF-8 디코딩하며 'data: {...}' 형식의 SSE 라인을
  줄 단위로 파싱하여 JSON.parse()로 이벤트를 추출한다.
  AbortController의 signal을 fetch에 전달하여 컴포넌트 언마운트 시
  스트림을 안전하게 정리(cleanup)할 수 있도록 구현하였다.

  이벤트 처리 분기
  수신된 이벤트 type에 따라 다음과 같이 처리한다.
    - ALERT 이벤트: sonner 라이브러리로 우측 상단 5초 토스트 알림,
      Web Audio API로 880Hz/660Hz 사인파 경보음 합성 재생,
      브라우저 Notification API로 OS 수준 알림 표시,
      TanStack Query의 invalidateQueries로 관련 캐시 즉시 무효화.
    - RISK_SCORE 이벤트: 해당 환자의 대시보드 및 환자 목록 캐시 무효화.
    - PING / CONNECTED: 재연결 카운터 리셋 또는 무시.

  연결 안정성
  네트워크 장애나 서버 재시작으로 연결이 끊길 경우, 지수 백오프
  알고리즘으로 자동 재연결을 시도한다.
  재연결 지연 시간: [1000, 2000, 5000, 10000, 30000] ms 순서로 증가하며,
  마지막 30초 간격에서 고정 반복하여 서버 복구 후 자동으로 연결을 복원한다.

■ 기술 2 — WebRTC WHEP 프로토콜 기반 실시간 영상 스트리밍

  배경 및 선택 이유
  독거노인의 실시간 상황을 보호자가 영상으로 직접 확인하기 위해
  저지연 영상 스트리밍 기능이 필요하였다. 기존 HLS/DASH 방식은
  수 초~수십 초의 지연이 발생하므로 응급 상황 대응에 부적합하다.
  mediamtx 미디어 서버가 지원하는 WHEP(WebRTC-HTTP Egress Protocol)을
  활용하여 브라우저에서 직접 WebRTC 연결을 수립하는 방식을 채택함으로써
  서버-브라우저 간 저지연 영상 전달을 실현하였다.

  구현 내용
  브라우저의 RTCPeerConnection을 생성하고 video/audio 방향을 recvonly로
  설정한 뒤, SDP Offer를 생성하여 mediamtx의 WHEP 엔드포인트에
  HTTP POST로 전송한다. 이때 SDP 내 코덱 우선순위를 H264로 재정렬하여
  mediamtx의 libx264 인코더와 협상이 원활하게 이루어지도록 하였다.
  서버의 SDP Answer를 받아 setRemoteDescription()을 호출하면
  WebRTC 미디어 채널이 개설되고 <video> 태그에 스트림이 연결된다.

  상태 관리 및 재연결
  VideoPlayer 컴포넌트는 세 가지 상태(connecting / live / error)를
  관리한다. connectionState 이벤트를 감지하여 'connected' 시 영상 재생을
  시작하고, 'failed' 또는 'disconnected' 상태가 되면 5초 후 RTCPeerConnection을
  재생성하여 자동 재연결을 시도한다. 컴포넌트 언마운트 시에는
  peerConnection.close()로 리소스를 해제한다.

  행동 레이블 오버레이
  VideoPlayer 위에 ActionOverlayPanel을 오버레이하여 AI가 분석한
  현재 행동 레이블(한국어)과 위험도(NORMAL/ABNORMAL/DANGER)를
  실시간으로 표시한다. 행동 레이블이 변경될 때마다 300ms opacity
  fade 애니메이션으로 자연스럽게 전환되며, DANGER 상태에서는
  pulse 애니메이션으로 위험 상황임을 즉각 시각화한다.

■ 기술 3 — React 19 SPA + TanStack Query v5 서버 상태 관리

  배경 및 선택 이유
  복수의 환자 데이터, 이벤트 이력, 대시보드 통계 등 다양한 서버
  데이터를 효율적으로 가져오고 화면과 동기화하기 위해 TanStack Query v5를
  도입하였다. useState + useEffect 조합으로 직접 폴링을 관리하면
  로딩/에러 상태, 캐시 무효화, 중복 요청 방지 등을 모두 수동으로
  처리해야 하지만, TanStack Query는 이를 선언적으로 처리한다.

  구현 내용
  모든 서버 데이터 조회는 커스텀 훅 형태로 추상화하였다.
    - usePatientList(): 60초 간격 환자 목록 폴링
    - useDashboard(patientId): 15초 간격 환자별 대시보드 폴링
    - useAllEvents(limit): 15초 간격 전체 이벤트 목록 폴링
    - usePatientEvents(patientId): 15초 간격 특정 환자 이벤트 폴링
    - useAlerts(): 20초 간격 알림 이력 폴링
  SSE로 ALERT 또는 RISK_SCORE 이벤트를 수신하면 관련 쿼리의
  invalidateQueries를 즉시 호출하여 백그라운드에서 갱신을 트리거한다.
  이를 통해 폴링 주기와 무관하게 서버 이벤트 발생 즉시 화면이 갱신된다.

  전역 에러 처리
  QueryCache의 onError 핸들러를 통해 모든 API 오류를 전역에서 처리한다.
  401 응답 시 인증 정보 삭제 후 /login 리다이렉트, 네트워크 오류 및
  서버 오류 시 소너 토스트 알림을 표시한다. 개별 컴포넌트에서
  중복 에러 처리 코드 없이 일관된 에러 UX를 제공한다.

■ 기술 4 — JWT 기반 인증 + 역할별 라우트 접근 제어 (UI 및 시각화)

  배경 및 선택 이유
  본 시스템은 admin(관리자)과 guardian(보호자) 두 가지 역할이 존재한다.
  역할에 따라 접근 가능한 페이지와 표시되는 기능이 달라야 하므로,
  JWT 토큰 기반 인증과 역할 기반 라우트 보호 컴포넌트를 설계하였다.

  구현 내용
  로그인 시 서버로부터 받은 access_token과 user 객체를 Zustand
  auth-store에 저장하고, persist 미들웨어로 localStorage에 자동
  직렬화하여 새로고침 이후에도 로그인 상태를 유지한다.
  모든 API 요청은 apiFetch() 함수를 통해 Authorization 헤더를
  자동 첨부하며, 401 응답 수신 시 즉시 인증 정보를 삭제하고
  /login으로 리다이렉트한다.
  AuthGuard 컴포넌트는 인증되지 않은 사용자가 보호 경로에 접근할 때
  /login으로 강제 이동시키며, RoleGuard 컴포넌트는 역할 불일치 시
  각 역할의 기본 페이지로 리다이렉트한다.
  admin은 관리자 대시보드(/admin), 환자 등록(/register-patient),
  응급 연락 버튼 등 추가 기능에 접근 가능하며, guardian은
  모니터링 뷰(/dashboard, /alerts, /profile)만 이용할 수 있다.

2.4  졸업 작품 최종 예상 목표 (프론트엔드)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 1) 프로젝트 목표

  프론트엔드의 최종 목표는 독거노인의 안전 상태를 보호자와 관리자가
  웹 브라우저에서 실시간으로 모니터링할 수 있는 단일 페이지
  애플리케이션(SPA) 대시보드를 구현하는 것이다.

  구체적인 목표는 다음과 같다.

  · Raspberry Pi 5의 AI가 생성하는 행동 분석 이벤트(낙상, 무활동,
    정상 활동 등)를 SSE 실시간 스트리밍으로 수신하여 보호자 화면에
    지연 없이 알림으로 전달해야 한다.
  · WebRTC WHEP 프로토콜을 통해 mediamtx 서버에서 전송하는 실시간
    영상을 브라우저에서 저지연으로 재생하고, 영상 위에 AI 분석 결과
    (행동 레이블, 위험도)를 오버레이로 표시해야 한다.
  · JWT 기반 인증과 역할 기반 접근 제어(admin/guardian)를 구현하여
    각 사용자가 자신의 권한 범위 내 기능만 이용할 수 있어야 한다.
  · TanStack Query로 서버 상태를 관리하고, 폴링과 SSE 기반 캐시
    무효화를 결합하여 화면이 항상 최신 데이터를 반영해야 한다.
  · 개별 환자 상세 페이지에서 위험점수 SVG 게이지, 이벤트 이력,
    7일 분석 차트(바차트/시간대별 활동/히트맵)를 통해 어르신의
    건강 상태 변화를 시각적으로 파악할 수 있어야 한다.

■ 2) 최종 결과물

  로그인 및 인증 시스템
  - 이메일·비밀번호 입력 폼(zod 유효성 검사, react-hook-form 연동)
  - JWT 토큰 localStorage 영속, 새로고침 후 자동 로그인 유지
  - AuthGuard: 비인증 접근 시 /login 리다이렉트
  - RoleGuard: admin → /admin, guardian → /dashboard 역할별 분기

  메인 대시보드 (/dashboard)
  - 전체 환자 카드 그리드(3열): 위험점수 게이지 바, 카메라 온라인
    상태 도트, 최근 이벤트 시각 표시
  - 통계 카드 3개(환자수/미확인 이벤트/위험 이벤트) — 카운트업 애니메이션
  - 최근 활동 타임라인(오늘/이번주/전체 필터)
  - 전체 위험 상태에 따른 헤더 그라디언트 색상 변화

  환자 상세 페이지 (/dashboard/:patientId)
  - WebRTC WHEP 실시간 영상 플레이어(connecting/live/error 3단계 상태 UI)
  - ActionOverlayPanel: 행동 레이블·위험도를 영상 위 오버레이로 표시
  - SVG 반원 위험 게이지(RiskGauge) + 세그먼트 바(RiskSegmentBar)
  - 최근 이벤트 목록(스크롤, 위험도 색상 테두리)
  - 분석 차트 3탭: 7일 누적 바차트 / 시간대별 활동 그리드 / 7일 히트맵
  - PDF 출력 기능(window.print() + Print CSS)
  - 이전/다음 환자 내비게이션, 응급연락 버튼(admin 전용)

  알림 패널 및 알림 이력
  - 헤더 알림 벨: 24시간 이내 미확인 이벤트 수 배지 실시간 표시
  - AlertPanel: 우측 슬라이드 패널(전체/오늘/위험 필터, Escape 닫기)
  - /alerts 페이지: 날짜 그룹 이력 뷰 / 테이블 뷰 전환,
    개별·전체 읽음 처리(localStorage 영속)

  SSE 실시간 알림 시스템
  - fetch 스트리밍 기반 /api/v1/alerts/stream 연결(인증 헤더 포함)
  - ALERT 이벤트: 소너 토스트 + Web Audio API 경보음 + 브라우저 알림
  - 지수 백오프 자동 재연결(최대 30초 간격)
  - 브라우저 탭 타이틀에 미확인 수 표시: '(N) 어르신 안전 돌봄 서비스'

  관리자 대시보드 (/admin, admin 전용)
  - slate-950 다크 테마, 환자 위험점수 내림차순 정렬 테이블
  - 보호자 등록(이메일/비밀번호/연결 환자) + 삭제 confirm 모달
  - TanStack useMutation으로 등록·삭제 처리, 7일 이벤트 테이블

  환자 등록 3단계 위저드 (/register-patient, admin 전용)
  - 1단계(기본정보) → 2단계(기기정보) → 3단계(확인 및 제출)
  - 단계별 유효성 검사, 프로그레스 바 + 단계 인디케이터

■ 3) 학습 및 습득 기술

  WebRTC 및 실시간 통신 기술
  WebRTC WHEP 프로토콜의 SDP Offer/Answer 협상 과정을 직접 구현하면서
  WebRTC의 동작 원리와 미디어 협상 메커니즘을 실전으로 이해할 수 있음.
  SDP 내 코덱 우선순위 재정렬, ICE Gathering 비동기 타이밍 처리,
  RTCPeerConnection 상태 머신 기반 재연결 로직을 작성하면서
  저수준 WebRTC API 사용 역량을 갖출 수 있음.

  Server-Sent Events (SSE) 커스텀 구현
  EventSource API의 Authorization 헤더 제약을 극복하기 위해
  fetch Streaming API와 ReadableStream을 직접 다루는 방법을 익힐 수 있음.
  청크 기반 스트림 파싱, 버퍼링 처리, AbortSignal 기반 클린업,
  지수 백오프 재연결 패턴을 직접 구현하면서 웹 표준 스트리밍 API의
  실제 활용 방법을 습득할 수 있음.

  React 19 + TanStack Query 서버 상태 관리
  TanStack Query v5의 queryKey 설계, staleTime/refetchInterval 조합,
  SSE 이벤트 기반 invalidateQueries를 활용한 선언적 데이터 동기화 패턴을
  실무 수준으로 익힐 수 있음. 폴링과 push를 결합하여 최적의 실시간성을
  달성하는 아키텍처를 설계하는 역량을 갖출 수 있음.

  Zustand를 활용한 클라이언트 상태 관리
  Zustand persist 미들웨어를 통한 localStorage 자동 동기화,
  여러 스토어(auth/read/seen)를 역할별로 분리하여 관리하는 패턴,
  셀렉터 훅으로 불필요한 리렌더를 방지하는 방법을 학습할 수 있음.

  JWT 기반 인증 및 역할 기반 접근 제어 구현
  Bearer Token 인증 흐름(로그인 → 토큰 저장 → 헤더 첨부 → 401 처리)을
  직접 구현하면서 웹 인증의 표준 패턴을 이해할 수 있음. AuthGuard/RoleGuard
  컴포넌트 패턴으로 라우트 수준의 접근 제어를 구현하는 방법을 익힐 수 있음.

  Web Audio API 및 Notification API
  위험 이벤트 발생 시 프로그래밍으로 경보음을 합성하는 Web Audio API의
  AudioContext, OscillatorNode, GainNode 사용법을 습득할 수 있음.
  브라우저 Notification API의 권한 요청 흐름과 OS 수준 알림 표시 방법을
  실전으로 익힐 수 있음.

  Recharts를 활용한 데이터 시각화
  7일 누적 바차트, 24시간 활동 그리드, 7일 히트맵 등 다양한 형태의
  시각화를 Recharts 라이브러리로 구현하면서 데이터 집계 및 차트 설계
  역량을 기를 수 있음.

  TypeScript 실무 적용
  API 응답 타입(User, Patient, EventItem, ActionEvent 등) 정의,
  제네릭 기반 apiFetch<T>() 함수 설계, 커스텀 에러 클래스 계층 구조
  설계를 통해 TypeScript 타입 시스템을 실무 수준으로 활용하는
  역량을 습득할 수 있음.

■ 4-1. 활용방안

  독거노인 응급 대응 서비스
  보호자 또는 사회복지사가 PC 웹 브라우저에서 별도의 앱 설치 없이
  접속하여 여러 노인의 생활 상태를 한 화면에서 동시에 모니터링할 수
  있다. 낙상, 장기 무활동 등 위험 이벤트 발생 시 토스트 알림·경보음·
  브라우저 알림을 통해 즉각적으로 인지하고, 실시간 영상으로 현장 상황을
  직접 확인한 뒤 119 신고 등 조치를 취할 수 있다. 이를 통해 기존
  방문 돌봄 서비스의 시간·공간 공백을 보완하는 24시간 무인 원격 돌봄
  서비스로 활용될 수 있다.

  지자체 및 사회복지기관 관제 시스템
  admin 계정으로 복수의 보호자와 환자를 등록·관리할 수 있어,
  사회복지관이나 지자체 단위에서 관할 독거노인 전체를 한 시스템으로
  관리하는 중앙 관제 솔루션으로 활용할 수 있다. 관리자 대시보드에서
  환자별 위험점수를 내림차순 정렬하여 우선순위가 높은 어르신을 즉시
  확인하고, 보호자(자원봉사자, 사회복지사) 계정을 개별 환자에 연결하여
  분산 관리 체계를 구축할 수 있다.

  의료 및 원격 진료 연계
  환자 상세 페이지에서 7일 이벤트 분석 차트(바차트, 시간대별 활동 그리드,
  히트맵)와 PDF 출력 기능을 활용하여 정기 방문 의료인이나 원격 진료
  시스템에 활동 패턴 리포트를 제출하는 자료로 활용할 수 있다. 위험점수
  SVG 게이지와 세그먼트 바는 비전문가도 건강 위험 수준을 직관적으로
  파악할 수 있어 의사소통 보조 도구로 활용 가능하다.

  스마트홈·실버케어 플랫폼 확장
  현재 구현된 프론트엔드 SPA는 API 명세가 유지되는 한 어떤 백엔드
  서버·AI 모델과도 연동 가능한 독립적인 구조로 설계되어 있다.
  향후 다른 IoT 센서 데이터나 웨어러블 디바이스 데이터를 동일한
  SSE 이벤트 스트림으로 수신하면, 낙상 감지 외에도 심박수·혈압 등
  생체 데이터를 실시간으로 시각화하는 스마트홈 플랫폼으로 확장할 수
  있는 기반이 된다.

■ 4-2. 기대효과

  기술적 기대효과

  · SSE fetch 스트리밍 직접 구현을 통해, 기존 WebSocket이나 외부
    실시간 서비스(Firebase, Pusher 등) 없이도 JWT 인증이 적용된
    실시간 서버 푸시를 구현하는 방법을 확립하였다. 이는 인증이
    필요한 SSE 환경에서 범용적으로 재사용 가능한 기술 자산이 된다.

  · WebRTC WHEP 프로토콜의 브라우저 직접 구현 경험을 통해,
    플러그인이나 Flash 없이 순수 웹 표준만으로 저지연 영상 스트리밍
    솔루션을 구축하는 노하우를 축적하였다. mediamtx 오픈소스 서버와
    조합하면 별도의 유료 CDN·스트리밍 플랫폼 없이 라이브 영상
    서비스를 구현할 수 있음을 실증하였다.

  · TanStack Query의 폴링과 SSE 기반 invalidateQueries를 조합한
    하이브리드 데이터 동기화 패턴을 실제 프로덕션 수준의 코드로
    구현하였다. 이 패턴은 실시간성이 요구되면서도 일관성 있는 서버
    상태 관리가 필요한 모든 React SPA 프로젝트에 적용 가능한
    아키텍처 모범 사례로 활용될 수 있다.

  사회적 기대효과

  · 보호자가 24시간 현장에 있지 않아도 PC 웹 브라우저만으로
    독거노인의 안전 상태를 실시간으로 확인할 수 있어, 물리적 돌봄
    공백을 기술로 보완하는 사회안전망 확충에 기여할 수 있다.

  · 위험 이벤트 발생 즉시 세 가지 채널(토스트·경보음·브라우저 알림)을
    통해 보호자에게 전달되므로, 낙상 등 응급 상황 발생 후 발견까지의
    시간을 단축하여 골든타임 확보율을 높이는 데 기여한다.

  · 영상은 HTTPS 암호화 통신과 인증 토큰으로 보호되고,
    역할 기반 접근 제어(admin/guardian)로 데이터 접근 권한이
    엄격히 관리되므로, 프라이버시 우려를 최소화하면서 돌봄 서비스를
    제공하는 윤리적 설계를 실현한다.

  경제적 기대효과

  · React, Vite, TanStack Query, Zustand, Tailwind CSS, shadcn/ui,
    Recharts, sonner 등 모두 오픈소스 라이브러리를 사용하여
    프론트엔드 구축에 별도의 라이선스 비용이 전혀 발생하지 않는다.

  · 별도의 유료 실시간 서비스(Firebase Realtime Database, Pusher 등)나
    고가의 CDN 영상 서비스 없이 SSE + WebRTC WHEP 조합만으로
    실시간 데이터 및 영상 스트리밍을 구현하여 운영 비용을 최소화한다.

  · 정적 파일(HTML/JS/CSS) 형태의 SPA로 빌드되므로 AWS S3 + CloudFront
    또는 Nginx 정적 서빙만으로 배포가 가능하여 서버 운영 비용이 낮다.
